# 流水账日记 App — 产品需求文档（PRD）

**文档版本**：v1.0  
**产品代号**：FlowLog  
**目标平台**：Android / iOS（Flutter 或原生 Material Design 3）  
**商业模式**：订阅制（Freemium）  
**核心定位**：睡前 30 秒完成记录，零认知负担的个人时间追踪与轻量资产管理工具

---

## 1. 产品概述

### 1.1 一句话描述
一款基于 Material Design 3 的极简日记应用，用户通过自然语言“流水账”式输入，系统自动提取习惯打卡与消费账单，并支持通过标准化插件补充结构化数据，帮助用户在无压力状态下完成每日复盘。

### 1.2 核心价值主张
- **极速记录**：打开即写，无需选择分类、标签或模板，降低记录阻力。
- **自动结构化**：后台自动解析文本，将“习惯”与“账单”从流水账中抽离为可统计的数据。
- **插件化扩展**：对非自动识别的固定场景（如饮水、睡眠、体重），提供标准插件卡片供用户一键填写。
- **订阅增值**：基础记录永久免费，高级功能（无限插件、深度统计、云同步、自定义识别规则）通过订阅解锁。

## 2. 用户画像与核心场景

| 用户角色 | 典型场景 | 痛点 | 产品解法 |
|---|---|---|---|
| **职场新人** | 睡前回忆今天做了什么、花了多少钱 | 懒得打开记账/习惯App分类填写 | 一句话输入，自动提取账单与习惯 |
| **学生/创作者** | 通勤、排队时突然有灵感 | 打开笔记App太重，事后整理麻烦 | 打开即写，自动保存时间戳与位置 |
| **自律追踪者** | 每天需要记录体重、饮水、睡眠时长 | 多个App数据分散 | 标准插件统一入口，流水账中自动关联 |

## 3. 功能需求（Functional Requirements）

### 3.1 核心记录模块（Core Journal）

**FR-001 极速输入页（Home）**
- 打开App直接进入输入页，无首页加载、无列表干扰。
- 输入框常驻屏幕下半区（拇指热区），支持多行文本。
- **快捷操作栏**：
  - 插入当前时间戳（如 `21:30`）
  - 插入当前位置（可选，默认关闭）
  - 拍照/选图（仅作为当日附件，非正文）
- **自动保存**：输入停止 2 秒后自动本地保存为草稿，无需手动点击保存。
- **提交方式**：点击“完成”或返回/熄屏时自动归档为当日条目。

**FR-002 时间轴浏览（Timeline）**
- 按日期倒序排列，每日仅显示一条主卡片（当日所有记录的聚合）。
- 卡片内文本超过 3 行折叠，点击展开。
- 顶部显示当日自动提取的**结构化摘要**：习惯完成数、支出总额、收入总额。
- 支持日历视图快速跳转日期。

**FR-003 自然语言解析引擎（NLP Parser）**
- **触发时机**：用户提交文本后，本地轻量解析（优先）或云端解析（复杂规则）。
- **识别范围**：
  - **账单识别**：货币符号（¥/$/€）、数字 + 单位（元/块/美金）、收支关键词（“花了”“收入”“转账”“报销”）。
    - 示例输入：`中午花了25块买咖啡，晚上打车回家12.5元` → 提取两条支出：餐饮 ¥25，交通 ¥12.5。
  - **习惯识别**：预定义习惯库中的动词 + 量词。
    - 示例输入：`今天跑步5公里，喝水大概1500ml` → 提取：跑步 5km，饮水 1500ml。
- **置信度机制**：解析结果以标签形式悬浮在文本下方，用户可一键确认或驳回（驳回后数据不入库，但记录用户反馈用于优化）。

### 3.2 结构化数据模块（Structured Data）

**FR-004 习惯追踪（Habits）**
- 系统自动识别的习惯条目进入“待确认”列表，用户确认后计入连续打卡。
- 支持用户预设习惯词库（如“背单词”“冥想”“阅读”），提升识别准确率。
- 统计维度：周完成率、月热力图、最长连续天数。

**FR-005 轻量账本（Bills）**
- 自动提取的账单进入当日账单列表。
- 支持用户事后修改分类（餐饮/交通/购物/居住/医疗/其他）。
- 统计维度：日/周/月支出趋势图、分类占比环形图。
- **注意**：不做复杂预算管理、不做账户体系、不做报销流程，保持极简。

### 3.3 标准化插件系统（Plugins）

**FR-006 插件定义**
- 插件是用户主动填写的结构化卡片，用于补充自动识别无法覆盖的场景。
- **标准内置插件**（免费版限用 3 个，订阅版无限）：
  1. **睡眠插件**：就寝时间、起床时间、睡眠时长（自动计算）、睡眠质量（1-5 星）。
  2. **身体数据**：体重、体脂率、步数（支持 HealthKit / Google Fit 读取）。
  3. **心情天气**：当日心情（5 级表情）、天气（自动获取或手动选择）。
  4. **项目工时**：选择项目名 + 投入小时数（面向自由职业者）。
  5. **饮水记录**：快速 +200ml / +500ml 按钮，或自定义输入。
- **插件交互**：在输入页底部以横向可滑动卡片形式存在，点击展开填写，填写后数据与当日流水账条目绑定。

**FR-007 插件接口规范（预留扩展）**
- 定义标准插件接口 `IPluginCard`，后续第三方开发者或你自己可新增插件：
  - `String pluginId`：唯一标识
  - `String displayName`：显示名称
  - `List<Field> fields`：字段列表（支持类型：Number, String, Date, Enum, Boolean）
  - `String iconName`：Material Symbols 图标名
  - `void onSubmit(Map data)`：提交回调
- 插件数据以 JSON 形式存储，与当日日记条目关联（`entryId` 外键）。

### 3.4 订阅与付费模块（Subscription）

**FR-008 订阅层级设计**

| 功能 | 免费版（Free） | 订阅版（FlowLog Pro） |
|---|---|---|
| 流水账记录 | 无限条 | 无限条 |
| 本地存储天数 | 最近 90 天 | 永久 |
| 云同步 | ❌ | ✅（端到端加密） |
| 自动识别习惯/账单 | 每日最多 5 条 | 无限 |
| 标准插件使用数量 | 最多 3 个 | 无限 |
| 自定义习惯词库 | 3 个 | 无限 |
| 自定义账单分类 | ❌ | ✅ |
| 数据导出（CSV/PDF） | ❌ | ✅ |
| 主题颜色（Material You） | 基础 4 色 | 动态取色 + 全色板 |

**FR-009 订阅系统接口预留**
- **支付渠道抽象层**：定义 `BillingProvider` 接口，预留以下实现：
  - `GooglePlayBillingProvider`（Android）
  - `AppStoreBillingProvider`（iOS）
  - `StripeBillingProvider`（Web/桌面端未来扩展）
  - `ManualBillingProvider`（促销码/兑换码/内测赠送）
- **订阅状态管理**：
  - 本地缓存订阅状态（`isPro`, `expiryDate`），启动时校验。
  - 服务端需维护 `subscriptions` 表，支持收据校验与续期回调。
- **降级策略**：订阅到期后，超量数据（如第 4 个插件、90 天前记录）变为只读，不可新增/修改，但可查看和导出。

## 4. 非功能需求（Non-Functional Requirements）

### 4.1 UI/UX 规范
- **设计系统**：严格遵循 **Material Design 3（Material You）** 规范。
  - 使用官方组件：`NavigationBar`、`FloatingActionButton`、`Card`、`BottomSheet`、`Chips`。
  - 动态色彩：Android 12+ 支持壁纸取色（`DynamicColors`），iOS 端提供静态 Material 调色板。
- **动画原则**：所有转场动画时长 200-300ms，使用 `ease-in-out`；禁止阻塞操作的模态加载动画。
- **无障碍**：支持 TalkBack / VoiceOver，所有图标按钮带 `contentDescription`。

### 4.2 性能需求
- **冷启动时间**：从点击图标到输入框可输入 ≤ 800ms（中低端机型）。
- **解析延迟**：本地解析流水账文本 ≤ 150ms（100 字以内）。
- **存储占用**：安装包 ≤ 25MB；用户数据（纯文本 + 轻量结构化数据）年增长 ≤ 50MB。

### 4.3 安全与隐私
- **本地优先**：所有原始日记文本默认仅本地存储，不上传云端（除非用户开启云同步）。
- **自动识别数据**：习惯与账单结构化数据可本地存储，云端同步时采用 AES-256 加密。
- **生物识别锁**：支持指纹/面容解锁（可选开启）。

### 4.4 离线与同步
- 完全支持离线使用，有网时自动同步。
- 冲突解决策略：以最新编辑时间为准，云端保留历史版本（Pro 功能）。

## 5. 数据模型（Data Schema）

```typescript
// 核心日记条目
interface Entry {
  id: UUID;
  date: Date;           // 归属日期（允许补记昨天）
  createdAt: Timestamp;
  updatedAt: Timestamp;
  rawText: String;      // 原始流水账文本
  images: List<String>; // 本地图片路径
  location: GeoPoint?;  // 可选位置
}

// 自动解析出的结构化数据（ habits / bills ）
interface ExtractedData {
  id: UUID;
  entryId: UUID;
  type: "habit" | "bill";
  rawSegment: String;   // 原文本片段（如"跑步5公里"）
  parsedValue: Number;  // 数值（5 或 25.0）
  unit: String;         // "km", "元", "ml", "min"
  category: String;     // 习惯名 或 账单分类
  confidence: Float;    // 解析置信度 0.0-1.0
  isConfirmed: Boolean; // 用户是否确认
}

// 插件数据
interface PluginRecord {
  id: UUID;
  entryId: UUID;
  pluginId: String;     // 如 "sleep", "weight"
  data: JSON;           // 插件自定义字段值
}

// 订阅（服务端）
interface Subscription {
  userId: UUID;
  tier: "free" | "pro";
  provider: "google_play" | "app_store" | "stripe" | "manual";
  receiptToken: String;
  startedAt: Timestamp;
  expiresAt: Timestamp;
  autoRenew: Boolean;
}
```

## 6. 技术架构建议

- **前端**：Flutter（Dart）或 Jetpack Compose（Android）/ SwiftUI（iOS）。
  - 推荐 **Flutter + Material 3 组件库**，一套代码双端，且 Google 官方维护 `flutter/material` 库。
- **本地存储**：SQLite（`sqflite` / `room`）存储文本与结构化数据；图片存本地文件系统，数据库仅存路径。
- **解析引擎**：
  - 第一阶段：本地正则 + 规则引擎（Dart/Java/Kotlin），覆盖 80% 常见场景。
  - 第二阶段：接入轻量云端 NLP（如 Google ML Kit 本地实体识别，或自建后端调用 LLM API）。
- **后端（云同步 + 订阅校验）**：
  - Node.js / Go 轻量服务，仅负责用户认证、订阅状态校验、加密数据同步。
  - 不处理原始日记内容（端到端加密，服务端无密钥）。

## 8. 待确认事项（需要你在开发前拍板）

1. **跨平台策略**：是否用 Flutter 一套代码，还是原生 Android/iOS 分开开发？（影响订阅接口的实现方式）
2. **自动识别范围**：账单是否只支持人民币？习惯词库是否允许用户自定义，还是仅系统预设？
3. **云同步方案**：是否自建后端，还是先用 Firebase / Supabase 等 BaaS 快速验证？
4. **订阅价格锚点**：Pro 版月费/年费定价区间（建议早期内测阶段暂不收费，仅预留接口，等用户量过万再开启付费）。
